## <img src="https://img.shields.io/badge/-About-8B5CF6?style=flat-square&logoColor=white" />

I'm a first-year Master's student in **Systems Security & Big Data (SSI)** at FST Tanger, focused on the intersection of defensive security operations and the data engineering that makes them work. My day-to-day is split between coursework, building real lab infrastructure, and competing in CTFs with **Team ANTISITBD**.

What I actually do, not just claim:

- Design and operate **multi-zone network defenses** in EVE-ng — Cisco ZPF, ZTNA posture checks, and ML-augmented IDS — under the supervision of Pr. Ikram Ben Abdel Ouahab.
- Build **stream-first detection pipelines** on Kafka + Spark Structured Streaming with PostgreSQL persistence and Streamlit visualization.
- Work the full **SOC analyst stack**: log ingestion, correlation, triage, and incident response with Splunk, Wazuh, Suricata, and TheHive.
- Lead the **IT GEEKS Club** at FST Tanger for the 2025–2026 mandate.

**Open to:** PFA internship as a **Junior SOC Analyst** or **Cloud Security Engineer** — Tanger or Casablanca, starting Spring 2026.

## <img src="https://img.shields.io/badge/-Featured%20Projects-8B5CF6?style=flat-square&logoColor=white" />

<details>
<summary><b>🛡️ &nbsp; Active Defense Architecture — Self-Healing Security Infrastructure</b> &nbsp; <img src="https://img.shields.io/badge/-In%20development-22C55E?style=flat-square" /></summary>

<br/>

A four-phase enterprise-grade defense architecture I'm building under Pr. Ikram Ben Abdel Ouahab at FST Tanger. The thesis: a network should *react and recover* on its own when something breaks the policy — not wait for an analyst to read the email.

The topology runs in EVE-ng inside VMware Workstation, with a Cisco vIOS perimeter firewall enforcing Zone-Based Policy Firewall rules, a ZTNA posture orchestrator written in Python that gates access by device state, a Snort-based IDS that generates new rules from observed traffic, and a Byzantine fault-tolerant failover layer so a single compromised node can't poison the consensus.

<table>
<tr><td><b>Stack</b></td><td>EVE-ng, Cisco vIOS, Python, Snort, VMware Workstation</td></tr>
<tr><td><b>Topology</b></td><td>7 nodes across 4 bridge networks (DMZ, Inside, Quarantine, Outside)</td></tr>
<tr><td><b>Key ideas</b></td><td>Zone-Based Policy Firewall · ZTNA posture checks · ML-driven IDS rule generation · BFT failover</td></tr>
<tr><td><b>Supervisor</b></td><td>Pr. Ikram Ben Abdel Ouahab — FST Tanger</td></tr>
<tr><td><b>Status</b></td><td>🟢 In active development</td></tr>
</table>

</details>

<summary><b>📡 &nbsp; Real-Time Anomaly Detection Pipeline (Kappa Architecture)</b> &nbsp; <img src="https://img.shields.io/badge/-Delivered-6D28D9?style=flat-square" /></summary>

<br/>

A stream-first security analytics pipeline. Logs land in Kafka, get processed by Spark Structured Streaming with both statistical (μ + 2σ thresholding) and ML-based classifiers, persist to PostgreSQL, and surface in a Streamlit operator console. Everything is containerized so the whole stack comes up with one `docker compose up`.

<tr><td><b>Architecture</b></td><td>Kappa (single streaming path, no batch layer)</td></tr>
<tr><td><b>Detection</b></td><td>μ + 2σ baseline + supervised classifier ensemble</td></tr>
